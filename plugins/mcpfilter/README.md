# MCP Filter — Улучшатель запроса для Bifrost

Плагин для Bifrost, который перед отправкой запроса в LLM генерирует улучшенное системное сообщение. Сообщение пользователя остаётся без изменений.

## Как работает

1. Клиент отправляет запрос в Bifrost
2. `PreLLMHook` читает последнее сообщение пользователя (read-only)
3. Отправляет его в LLM с `system_prompt` (улучшающая инструкция)
4. Если LLM ответил вовремя — устанавливает/заменяет системное сообщение в запросе
5. Если таймаут или ошибка — запрос проходит как есть, без изменений

**Плагин не меняет сообщение пользователя и модель.** Он только генерирует и вставляет системное сообщение.

## Установка

```bash
make plugin
```

Через Bifrost Web UI: Plugins → Add Plugin → путь к `mcpfilter.so`

## Конфигурация

```json
{
  "enabled": true,
  "system_prompt": "Проанализируй запрос пользователя и сформулируй системное сообщение, которое поможет LLM дать максимально точный ответ. Отвечай только системным сообщением без пояснений.",
  "model_override": "gpt-4o-mini",
  "timeout_seconds": 10,
  "provider_base_url": "https://api.openai.com/v1",
  "provider_api_key": "env.OPENAI_API_KEY"
}
```

| Поле | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `enabled` | bool | `false` | Включить плагин |
| `system_prompt` | string | `""` | Инструкция для LLM-улучшения |
| `model_override` | string | `""` | Модель для улучшения (пусто = модель из запроса) |
| `timeout_seconds` | int | `10` | Максимум времени на улучшение |
| `provider_base_url` | string | `https://api.openai.com/v1` | URL API провайдера |
| `provider_api_key` | string | `""` | API-ключ (`env.VAR` для переменных окружения) |

## Что происходит с системным сообщением

- Если в запросе уже есть системное сообщение — плагин **заменяет** его содержимое
- Если системного сообщения нет — плагин **добавляет** его в начало запроса

Сообщение пользователя всегда остаётся без изменений.

## Отмена при отключении клиента

1. Клиент нажимает «Отмена» (Copilot)
2. `BifrostContext.Done()` срабатывает
3. HTTP-запрос к улучшителю прерывается мгновенно
4. Оригинальный запрос проходит без изменений, токены не потрачены

## Сборка

```bash
make plugin          # для текущей архитектуры
make plugin-docker   # через Docker (для NAS)
```
